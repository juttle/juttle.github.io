Write once, run against anything

Find all the log lines with status_code 500 in the last day and return the per-minute counts - either from splunk, a file, or elastic:

read -last :1 day: status_code = 500 | reduce -every :1m: count()

Moving averages are just one line away

Return the 1 hour moving average CPU usage, updated every minute:

read -last :6 hours: name = 'cpu' | reduce -over :1h: -every :1m: avg('value') by host

Juttle Service

Juttle Service exposes a REST API to run juttles. Hit it directly, or use Juttle Client Library to make streaming juttles and charts a breeze.

Embedded Juttle

Embedded juttle lets you run juttle queries against in-memory data instead of data stores, right in the browser.

  • Code
  • Result
const points = [
    {
        time: new Date(1000),
        v: 1
    },
    {
        time: new Date(2000),
        v: 2
    }
];

let juttle = new EmbeddedJuttle('put v2 = v | view table');

juttle.run({
    wait: true,
    points: points
}).then((result) => {
    console.log(JSON.stringify(result, null, 2));
})
.catch((err) => {
    console.log("got an error trying to do runBatch: " + err);
});
